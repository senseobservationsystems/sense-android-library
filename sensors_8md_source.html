3 # Sense Sensor {#sense_sensor}
4 
5 Almost all sensors in the Sense Library implement the SenseSensor interface. This interface provides several functions that are common for all sensors:
6 * start and stop sensing
7 * get and set sample rate
8 
9 The SenseSensor interface is defined in nl.sense_os.service.shared.SenseSensor
10 
11 # Base Sensor {#base_sensor}
12 
13 BaseSensor is a base implementation of [SenseSensor](##SenseSensor). It inherits from nl.sense_os.service.subscription.BaseDataProducer and therefore it also implements nl.sense_os.service.subscription.DataProducer. It provides a basic implementation to set and get sample rate, and leaves the start and stop sensing mechanism to the specific sensor it inherits.
14 
15 BaseSensor is implemented in nl.sense_os.service.subscription.BaseSensor
16 
17 # Periodic Polling Sensor {#periodic_polling_sensor}
18 
19 This is an interface that all periodic-based sensors should implement. It provides an **isActive** method for [PeriodicPollAlarmReceiver](##PeriodicPollAlarmReceiver) to check if the sensor is still active, and a **doSample** method which [PeriodicPollAlarmReceiver](##PeriodicPollAlarmReceiver) will call to do the actual sampling at a scheduled time.
20 
21 The PeriodicPollingSensor interface is defined in nl.sense_os.service.shared.PeriodicPollingSensor
22 
23 # Periodic Poll Alarm Receiver {#periodic_poll_alarm_receiver}
24 
25 This is a generic BroadcastReceiver for periodic based sensors. Every periodic-based sensor should have an instance of this class and start it when it starts sensing. This instance will register itself to [Scheduler](##Scheduler) to be run at a specific rate based on the interval value of the specific sensor (see nl.sense_os.service.subscription.BaseSensor.sampleDelay).
26 
27 It will call the **doSample** method of the sensor instance it is bound to when it is scheduled to be run.
28 
29 PeriodicPollAlarmReceiver is implemented in nl.sense_os.service.shared.PeriodicPollAlarmReceiver

68 # Sensor Data Structure {#sensor_data_structure}
69 
70 Every datapoint from a sensor should be encapsulated as nl.sense_os.service.shared.SensorDataPoint object. It contains several fields corresponding to data point fields at CommonSense:
71 * sensor name
72 * sensor description (corresponds to sensor_type field in CommonSense)
73 * timestamp
74 * data type
75 * value
76 
77 Sensor datapoints can be one of the following types:
78 * Integer
79 * Float
80 * Bool
81 * Double
82 * String
83 * ArrayList
84 * JSON
85 * JSONString
86 * File
87 * SensorEvent
88 
89 Here is an example of how to create a new datapoint :
90 
91 ~~~
92 SensorDataPoint dataPoint = new SensorDataPoint(value);
93 dataPoint.sensorName = sensorName;
94 dataPoint.sensorDescription = sensor.getName();
95 dataPoint.timeStamp = SNTP.getInstance().getTime();
96 ~~~
97 
98 This sensor datapoint also needs to be sent to [MsgHandler](documentation/msg_handler.md) so it can be stored in LocalStorage and later in CommonSense. Here is an example of how to send a sensor datapoint to be stored by MsgHandler.
99 
100 ~~~
101 Intent sensorData = new Intent(context.getString(R.string.action_sense_new_data));
102 sensorData.putExtra(DataPoint.SENSOR_NAME, SensorNames.NOISE);
103 sensorData.putExtra(DataPoint.VALUE, BigDecimal.valueOf(dB).setScale(2, 0).floatValue());
104 sensorData.putExtra(DataPoint.DATA_TYPE, SenseDataTypes.FLOAT);
105 sensorData.putExtra(DataPoint.TIMESTAMP, startTimestamp);
106 sensorData.setPackage(context.getPackageName());
107 context.startService(sensorData);
108 ~~~
